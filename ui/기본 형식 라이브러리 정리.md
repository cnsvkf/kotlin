# 기본 구조

```kotlin
package com.example.kt37

import android.os.Bundle
import androidx.activity.ComponentActivity
import androidx.activity.compose.setContent
import androidx.compose.material3.Text
import androidx.compose.runtime.Composable
import androidx.compose.ui.tooling.preview.Preview

class MainActivity : ComponentActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContent {
            FirstScreen()
        }
    }
}

@Composable
fun FirstScreen() {
    Text(text = "드디어 첫 UI 성공!")
}

@Preview(showBackground = true)
@Composable
fun DefaultPreview() {
    FirstScreen()
}
```

### @Composable 기능
1. Composable 함수끼리만 호출 가능하게 만든다
2. ___상태 변화 시 자동으로 다시 실행됨___

---
```kotlin
import androidx.compose.runtime.*
```


- ___import androidx.compose.runtime.Composable
기능 포함 + 그 이외의 기능___
    - @Composable

- remember :상태 유지

->
```kotlin
val count = remember { 0 }// recomposition 되어도 값 유지(UI 재실행)

val count = rememberSaveable { 0 } //화면 회전, 재생성에도 유지(상태 유지 레벨이 더 강함)
```

- State : __상태 관리__
   
-> 
   
 
 ```kotlin
    var count by remember { mutableStateOf(0) } //UI 자동갱신 + remember로 값 유지

    val count: State<Int>// 읽기 전용

    val count: MutableState<Int>// 변경 가능
 ```
- derivedStateOf (계산 상태)
    
- Side Effect (UI 외 작업)

->

```kotlin
LaunchedEffect(Unit) {
    // 화면 시작 시 실행(+비동기 작업 : 동시에 실행되는 작업)
}

DisposableEffect(Unit) {
    onDispose {
        // 정리 작업
    }
}
SideEffect {
    // recomposition 후 실행
}
```

- key : 재구성 제어, 특정 값 기준으로 UI 다시 생성

->
``` kotlin
key(값) {
    // key(값)이 변경 될 시 초기화 후 실행 아닐 시 그냥 유지
}
```

- rememberCoroutineScope :  Composable 안에서 코루틴(비동기 작업 쉽게) 실행 가능

->

``` kotlin
val scope = rememberCoroutineScope()
scope.launch {
    // 비동기 작업
}
``` 

- CompositionLocal 

->

``` kotlin
val LocalContext = compositionLocalOf { ... } // Compose 하위 UI 어디서든 꺼내 쓸 수 있는 공유 값 저장소 키(이름 표) 를 만듦

// 값 담기
CompositionLocalProvider(
    LocalContext provides "홍길동"
) {
    MyScreen()
} // LocalContext 라는 키에 홍길동 값 담음
```
