- try { } : 에러 발생 코드도 일단 실행
- catch { } : 에러 캐치
- finally { } : 무조건 실행

    - 예시

```kotlin
fun main() {
    try {
        val num = "abc".toInt()
        println(num)
    } catch (e: NumberFormatException) {
        println("숫자 변환 실패")
    } finally {
        println("무조건 실행")
    }
}
/* 결과
숫자 변환 실패
무조건 실행 */
```

---
- throw : 강제 에러
    
    - 형식
```kotlin
fun checkAge(age: Int) {
    if (age < 0) {
        throw IllegalArgumentException("나이는 0보다 작을 수 없음")
    }
}

fun main() {
    try {
        checkAge(-1)
    } catch (e: IllegalArgumentException) {
        println(e.message)
    }
} // 결과 : 나이는 0보다 작을 수 없음
```

---

- ?. : null이면 실행 안 하고 null 반환

- ? : null 가능

- ?: : 왼쪽이 null이면 오른쪽 실행
    

- as? : 타입 변환, 실패 시 null 
    
    - 형식

```kotlin
fun main() {
    val value: Any = "hello"
    val result: Int? = value as? Int
    println(result)
} // null
```

- !! : 무조건 Null 아님을 확신(Null이면 터짐)
    - __별로 안씀__
    - 형식
```kotlin
fun main() {
    val name: String? = "철수"
    println(name!!.length)
}
```
