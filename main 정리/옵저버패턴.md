> ## 구조
- Subject의 상태가 바뀌면 Subject가 리스트에 추가된 저장된Obsever의 기능을 씀
    - 기본구조
```kotlin
interface Observer {
    fun update(message: String)
}

interface Subject {
    // 옵저버 추가
    fun addObserver(observer: Observer)
    // 옵저버 삭제
    fun removeObserver(observer: Observer)
    // 알림
    fun notifyObservers()
}
```
---
> ## 예시

```kotlin
interface Observer {
// subject에게 알림 받을 시 할 행동
    fun update(message: String)
}

class UserObserver(private val name: String) : Observer {
    override fun update(message: String) {
        println("$name 알림 받음: $message")
    }
}

class EventSubject {
    private val observers = mutableListOf<Observer>()
    private var eventMessage = ""

    fun addObserver(observer: Observer) {
        observers.add(observer)
    }

    fun removeObserver(observer: Observer) {
        observers.remove(observer)
    }
// 이벤트 함수
    fun changeEvent(message: String) {
        eventMessage = message
        notifyObservers()
    }

// 모든 옵저버에게 알림
    private fun notifyObservers() {
        observers.forEach { it.update(eventMessage) }
    }
}

fun main() {
    val subject = EventSubject()

    val user1 = UserObserver("철수")
    val user2 = UserObserver("영희")

    subject.addObserver(user1)
    subject.addObserver(user2)

// 이벤트 발생
    subject.changeEvent("새 공지 등록")
}
```

-> message 매개변수가 전달받기에 공용으로 사용하는 것처럼 보임
